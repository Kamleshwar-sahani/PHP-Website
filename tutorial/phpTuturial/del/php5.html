PHP constants

In PHP, constants are identifiers (names) that hold values that cannot be changed during the execution of a script. Once defined, the value of a constant remains the same throughout the script's execution. Constants are useful for storing fixed values that should not be altered, such as configuration settings or mathematical constants.

Here's an overview of how constants work in PHP:

Naming Convention: By convention, constant names are typically written in uppercase letters with underscores between words (snake_case). For example: PI, APP_NAME, DB_HOST.

  1. Constants are Global: Once defined, constants are available in all scopes throughout the script, including within functions, classes, and included files.
  2. Predefined Constants: PHP also provides several predefined constants that are available for use without any additional setup. These constants represent information about the PHP environment, such as PHP version, operating system, and server details. Some examples of predefined constants are PHP_VERSION, PHP_OS, and __FILE__.
  3. Magic Constants: PHP has a set of special constants called magic constants that provide information about the script's execution environment. These constants start with two underscores (__). Some common magic constants are __LINE__, __FILE__, __DIR__, and __FUNCTION__.
  4. Constants vs. Variables: Unlike variables, constants cannot be reassigned or modified after they are defined. They are immutable throughout the script's execution.

Using constants helps make code more readable, maintainable, and less prone to accidental modifications. They provide a way to define meaningful names for fixed values and avoid using "magic numbers" or hard-coded values throughout the code.

Note: Starting from PHP 7.0, you can also define scalar constants with a scalar data type, such as const INTEGER_CONSTANT = 42; or define('STRING_CONSTANT', 'Hello');.
